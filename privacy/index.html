Privacy Policy
This document is the privacy policy of PenCake operated by Diffathy Inc. Updated: 29 December 2020
User-Generated Content
All user-generated content in PenCake is stored only on the user's device. PenCake does not transfer user-generated content to or store on external servers. PenCake does not operate a separate server to store user-generated content. Therefore, Diffathy Inc does not own or collect user-generated content.
When Using the Synchronization Function
When using the sync function provided by PenCake, content generated by a user is not only stored on the user's device but also stored in the third-party cloud storage of the user's own account. No user content is stored anywhere else.
If a user has multiple devices and each device has a sync connection to the same account's cloud storage, the same user content is stored on all devices.
Cloud storages that can be synced with PenCake include iCloud Drive and Google Drive. The privacy policy related to each cloud storage is attributed to the privacy policy under the agreement between each service provider and user.
Google User Identifier
When a user logs in to Google Drive to set up synchronization, Google collects the user identifier to manage OAuth authentication. The official guide from Google in this regard can be found in this document.
Anonymized Product Usage Information
PenCake uses Firebase Analytics provided by Google for product improvement through user status analysis and product function usage analysis. It also uses Firebase Crashlytics to analyze abnormal app termination. And for remote configuration of the product, it uses Firebase Remote Config. These services collect anonymous product usage information that is not linked to user identity. We do not use this information for any other purpose other than those described above. For example, we do not use the collected information for targeted advertising purposes or provide it to third parties.
The information collected by each service is as follows.
Commonly collected information
Device model, OS, app bundle ID, development platform
Firebase Analytics
Product interaction events such as app launch, app update, and session duration.
App instance identifier, anonymized IP address, etc
Vendor identifier (iOS), Advertising ID (Android)
Firebase Crashlytics
Information related to crash errors (stack traces, relevant application state, custom keys, logs)
Firebase Remote Config
No additional information is collected.
For more details, please refer to the two articles provided by Google below.
https://firebase.google.com/docs/ios/app-store-data-collection
https://support.google.com/analytics/answer/10285841
