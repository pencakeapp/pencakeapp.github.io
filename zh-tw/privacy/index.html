隱私權政策
本文件是 Diffathy Inc. 運營的 PenCake 的隱私權政策。 更新日期：2020年12月29日
使用者生成內容
PenCake 中所有使用者生成的內容僅儲存在使用者的設備上。PenCake 不會將使用者生成的內容傳輸到或儲存在外部服務器上。PenCake 不運行單獨的服務器來儲存使用者生成的內容。因此，Diffathy Inc 不擁有或收集使用者生成的內容。
使用同步功能時
使用 PenCake 提供的同步功能時，使用者生成的內容不僅會儲存在使用者的設備上，還會儲存在使用者自己賬戶的第三方雲端儲存中。使用者內容不會儲存在其他任何地方。
如果使用者擁有多台設備，並且每台設備都與同一帳戶的雲端儲存同步連接，則相同的使用者內容將儲存在所有設備上。
可以與 PenCake 同步的雲端儲存包括 iCloud Drive 和 Google Drive。各雲端儲存相關的隱私權政策歸屬於各服務提供商與使用者之間協議下的隱私權政策。
Google 使用者識別碼
當使用者登錄 Google Drive 設定同步時，Google 會收集使用者識別碼來管理 OAuth 身份驗證。Google 在這方面的官方指南可以在這個文件中找到。
匿名產品使用資訊
PenCake 使用 Google 提供的 Firebase Analytics 通過使用者狀態分析和產品功能使用分析來進行產品改進。它還使用 Firebase Crashlytics 來分析異常的應用程序終止。對於產品的遠程配置，它使用 Firebase Remote Config。這些服務收集與使用者身份無關的匿名產品使用資訊。我們不會將這些資訊用於上述目的以外的任何其他目的。例如，我們不會將收集到的資訊用於有針對性的廣告目的或將其提供給第三方。
每個服務收集的資訊如下。
共同收集的資訊
設備型號、作業系統、應用程式套件 ID、開發平台
Firebase Analytics
產品交互事件，例如應用程式啟動、應用程式更新和工作階段持續時間。
應用程式執行個體識別碼、匿名 IP 地址等
Vendor identifier (iOS), Advertising ID (Android)
Firebase Crashlytics
與當機錯誤相關的資訊 (stack traces, relevant application state, custom keys, logs)
Firebase Remote Config
不收集其他資訊。
更多詳情請參考 Google 提供的以下兩篇文章。
https://firebase.google.com/docs/ios/app-store-data-collection
https://support.google.com/analytics/answer/10285841
