隐私政策
本文档是 Diffathy Inc. 运营的 PenCake 的隐私政策。 更新日期：2020年12月29日
用户生成内容
PenCake 中所有用户生成的内容仅存储在用户的设备上。PenCake 不会将用户生成的内容传输到或存储在外部服务器上。PenCake 不运行单独的服务器来存储用户生成的内容。因此，Diffathy Inc 不拥有或收集用户生成的内容。
使用同步功能时
使用 PenCake 提供的同步功能时，用户生成的内容不仅会存储在用户的设备上，还会存储在用户自己账户的第三方云存储中。用户内容不会存储在其他任何地方。
如果用户拥有多台设备，并且每台设备都与同一帐户的云存储同步连接，则相同的用户内容将存储在所有设备上。
可以与 PenCake 同步的云存储包括 iCloud Drive 和 Google Drive。各云存储相关的隐私政策归属于各服务提供商与用户之间协议下的隐私政策。
Google 用户标识符
当用户登录 Google Drive 设置同步时，Google 会收集用户标识符来管理 OAuth 身份验证。Google 在这方面的官方指南可以在这个文档中找到。
匿名产品使用信息
PenCake 使用 Google 提供的 Firebase Analytics 通过用户状态分析和产品功能使用分析来进行产品改进。它还使用 Firebase Crashlytics 来分析异常的应用程序终止。对于产品的远程配置，它使用 Firebase Remote Config。这些服务收集与用户身份无关的匿名产品使用信息。我们不会将这些信息用于上述目的以外的任何其他目的。例如，我们不会将收集到的信息用于有针对性的广告目的或将其提供给第三方。
每个服务收集的信息如下。
共同收集的信息
设备型号、操作系统、应用捆绑包 ID、开发平台
Firebase Analytics
产品交互事件，例如应用启动、应用更新和会话持续时间。
应用实例标识符、匿名 IP 地址等
Vendor identifier (iOS), Advertising ID (Android)
Firebase Crashlytics
与崩溃错误相关的信息 (stack traces, relevant application state, custom keys, logs)
Firebase Remote Config
不收集其他信息。
更多详情请参考 Google 提供的以下两篇文章。
https://firebase.google.com/docs/ios/app-store-data-collection
https://support.google.com/analytics/answer/10285841
